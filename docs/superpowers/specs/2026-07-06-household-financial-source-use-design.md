# 가구원 금융수단 사용 허용 설계

## 목표

금융수단 소유자가 계좌와 결제수단별로 가구원 사용을 허용하고, 다른 가구원이 이를 공용 가계부 기록에서 선택할 수 있게 한다.

## 범위

- 계좌와 결제수단에 가구 전체 단위의 사용 허용 설정을 제공한다.
- 기존 금융수단과 신규 금융수단의 기본값은 사용 불가다.
- 허용된 금융수단은 공용 지출, 수입, 내부이체, 비지출 출금에서 사용할 수 있다.
- 개인 기록과 주식 거래에는 기존 소유권 규칙을 유지한다.
- 관리 화면의 금융수단 조회 범위와 기존 공용 기록 알림은 변경하지 않는다.
- 구성원별 권한과 별도 사용 알림은 추가하지 않는다.

## 데이터 모델

`accounts`와 `payment_methods`에 다음 열을 추가한다.

```sql
is_household_usable boolean not null default false
```

기존 행은 `false`로 마이그레이션된다. 설정은 현재와 미래의 모든 가구원에게 적용되며 금융수단 소유자만 변경할 수 있다. 별도 권한 테이블은 만들지 않는다.

## 서버 권한

기존 중앙 금융수단 검증 함수가 다음 규칙을 적용한다.

1. 금융수단이 같은 가구에 속해야 한다.
2. 기록 소유자가 금융수단 소유자면 개인·공용 기록 모두 허용한다.
3. 소유자가 다르면 기록이 공용이고 금융수단의 `is_household_usable`이 `true`인 경우만 허용한다.
4. 직접 API 요청도 같은 검증을 거치며 위반 시 `403`을 반환한다.

권한 검증이 성공한 뒤 기존 잔액 동기화 경로를 사용한다. 허용된 체크카드 사용은 연결 계좌가 별도로 허용되지 않았더라도 해당 계좌 잔액에 반영한다.

## 생성·수정·삭제

- 생성은 저장 직전에 현재 권한을 검증한다.
- 기존 기록의 잔액 효과가 바뀌는 수정은 현재 권한을 다시 검증한다.
- 사용 권한이 해제된 기존 기록은 제목, 날짜, 카테고리, 메모, 태그처럼 잔액 효과가 없는 정보만 수정할 수 있다.
- 기록 소유자는 권한 해제 후에도 기존 기록을 삭제하고 잔액 효과를 되돌릴 수 있다.
- 다른 사람의 금융수단을 사용해도 기록 소유권은 작성자에게 남는다. 금융수단 소유자는 기존 변경 요청 흐름을 사용한다.

## UI

계좌와 결제수단의 추가·수정 폼에 기본값이 꺼진 `가구원 사용 허용` 스위치를 표시한다.

> 공용 가계부 기록에서 모든 가구원이 사용할 수 있어요.

관리 목록에는 현재 사용 허용 상태를 배지로 표시한다. 관리 화면에서는 기존처럼 모든 가구원의 금융수단과 잔액을 조회할 수 있다.

금융수단 선택 목록은 기록 공개범위에 따라 다음처럼 필터링한다.

- 개인 기록: 본인 금융수단만 표시
- 공용 기록: 본인 금융수단과 사용 허용된 가구원 금융수단 표시

선택 항목과 확인 화면에는 `신한카드 · 홍길동`처럼 소유자 이름을 항상 표시한다. 작성 중 공용 기록을 개인 기록으로 바꿔 현재 선택이 무효가 되면 선택값을 지우고 다시 선택하게 한다.

## 테스트

- 금융수단 스키마와 API가 기본값, 생성, 수정 값을 보존하는지 검증한다.
- 본인/타인, 개인/공용, 허용/비허용 권한 조합을 검증한다.
- 체크카드의 연결 계좌 잔액 반영을 검증한다.
- 권한 해제 후 비금융 수정과 삭제는 허용하고 잔액 변경 수정은 거부하는지 검증한다.
- 선택 목록 필터링, 소유자 표시, 공개범위 변경 시 선택 초기화를 검증한다.
- 관리 폼의 스위치와 목록 배지를 검증한다.

## 제외

- 구성원별 사용 권한
- 금융수단 사용 전용 알림
- 금융수단 조회 제한
- 주식 거래 계좌 공유
