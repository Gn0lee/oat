# DESIGN.md

> UI/UX 원칙 및 디자인 가이드

## TL;DR

- **토스 스타일 참고** - 큰 숫자, 충분한 여백, 카드 기반, 직관적 흐름
- **데이터 중심** - 핵심 숫자가 가장 먼저 눈에 들어와야 함
- **반응형** - 모바일(하단 탭), PC(상단 네비게이션)
- **컬러** - 인디고 블루 기반, 상승 빨강/하락 파랑 (한국식)
- **다크모드** - MVP 제외, 2단계에서 추가

---

## 1. 디자인 원칙

### 1.1 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **데이터 중심** | 숫자와 핵심 정보가 가장 먼저 눈에 들어와야 함 |
| **단순함** | 한 화면에 하나의 목적, 불필요한 요소 제거 |
| **직관적 흐름** | 설명 없이도 다음 행동을 알 수 있어야 함 |
| **신뢰감** | 금융 서비스답게 안정적이고 깔끔한 인상 |

### 1.2 토스에서 배울 점

```
✅ 큰 숫자, 작은 레이블
✅ 충분한 여백 (빽빽하지 않게)
✅ 카드 기반 정보 그룹화
✅ 부드러운 전환 애니메이션
✅ 중요한 것만 색으로 강조
```

```

├─────────────────────────────────────────┤
│                                         │
│              Main Content               │
│                                         │
│  ┌─────────┐  ┌─────────┐              │
│  │  Card   │  │  Card   │  ...         │
│  └─────────┘  └─────────┘              │
│                                         │
├─────────────────────────────────────────┤
│  (Mobile Only) Bottom Navigation        │
└─────────────────────────────────────────┘
```

### 2.2 반응형 브레이크포인트

| 이름 | 크기 | 대상 |
|------|------|------|
| `sm` | 640px | 모바일 (기본) |
| `md` | 768px | 태블릿 |
| `lg` | 1024px | 작은 데스크톱 |
| `xl` | 1280px | 데스크톱 |

```typescript
// Tailwind 사용 예시
<div className="px-4 md:px-6 lg:px-8">
  <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
```

### 2.3 네비게이션

**모바일**: 하단 탭 네비게이션
```
[홈]  [분석]  [자산]  [설정]
```

**PC**: 사이드 네비게이션
```
┌──────────────┐
│ oat 로고      │
├──────────────┤
│ 홈           │ → /home
│ 분석         │ → /dashboard
│ 자산         │ → /assets
│ 설정         │ → /settings
└──────────────┘
```

**자산 페이지 내부**: 유형별 탭 또는 카드
```
/assets
├── 전체 보유 현황 → /assets/total/holdings
├── 주식/ETF      → /assets/stock/...
├── 현금 (준비중)  → /assets/cash/...
├── 부동산 (준비중) → /assets/real-estate/...
└── 기타 (준비중)  → /assets/other/...
```

## 3. 컬러

### 3.1 컬러 선정 원칙

| 원칙 | 설명 |
|------|------|
| 신뢰감 | 금융 서비스에 어울리는 안정적인 톤 |
| 현대적 | 올드하거나 투박하지 않은 느낌 |
| 절제 | 색상은 최소화, 강조점에만 사용 |

**추천 계열**
- 블루/인디고 계열: 신뢰, 안정
- 그린/틸 계열: 성장, 금융
- 퍼플 계열: 현대적, 프리미엄

### 3.2 컬러 구조

```css
:root {
  /* Primary - 인디고 블루 (신뢰감, 현대적) */
  --primary: #4F46E5;        /* Tailwind indigo-600 */
  --primary-dark: #4338CA;   /* indigo-700 */
  --primary-light: #EEF2FF;  /* indigo-50 */
  
  /* Neutral - 텍스트, 배경 */
  --gray-900: #191F28;       /* 메인 텍스트 */
  --gray-700: #4E5968;       /* 서브 텍스트 */
  --gray-500: #8B95A1;       /* 비활성 텍스트 */
  --gray-300: #D1D6DB;       /* 보더 */
  --gray-100: #F2F4F6;       /* 배경 */
  --white: #FFFFFF;
  
  /* Semantic - 의미 기반 */
  --positive: #F04452;       /* 상승 (빨강) */
  --negative: #3182F6;       /* 하락 (파랑) */
  --warning: #FF9F00;
  --success: #03B26C;
}
```

### 3.3 컬러 사용 규칙

| 용도 | 컬러 |
|------|------|
| 메인 텍스트 | `gray-900` |
| 보조 텍스트 | `gray-700` |
| 비활성/힌트 | `gray-500` |
| 주요 액션 버튼 | `primary` |
| 수익 상승 | `positive` (빨강) |
| 수익 하락 | `negative` (파랑) |
| 배경 | `gray-100` 또는 `white` |
| 카드 배경 | `white` |

### 3.4 다크모드 (2단계)

MVP에서는 라이트모드만 지원. 2단계에서 다크모드 추가 예정.

```css
/* 2단계에서 추가 */
[data-theme="dark"] {
  --gray-900: #FFFFFF;
  --gray-100: #191F28;
  /* ... */
}
```

## 5. 컴포넌트 가이드

### 5.1 카드

정보를 그룹화하는 기본 단위

```tsx
<div className="bg-white rounded-2xl p-5 shadow-sm">
  {/* 내용 */}
</div>
```

| 속성 | 값 |
|------|-----|
| 배경 | `white` |
| 모서리 | `rounded-2xl` (16px) |
| 패딩 | `p-5` (20px) |
| 그림자 | `shadow-sm` |

### 5.2 버튼

```tsx
// Primary
<Button className="w-full bg-primary text-white rounded-xl py-3">
  자산 추가하기
</Button>

// Secondary
<Button variant="outline" className="rounded-xl">
  취소
</Button>
```

| 타입 | 용도 |
|------|------|
| Primary | 주요 액션 (저장, 추가, 확인) |
| Secondary/Outline | 보조 액션 (취소, 뒤로) |
| Ghost | 덜 중요한 액션 |

### 5.3 입력 필드

```tsx
<div className="space-y-2">
  <label className="text-sm text-gray-700">종목명</label>
  <Input 
    className="rounded-xl border-gray-300 focus:border-primary"
    placeholder="종목을 검색하세요"
  />
</div>
```

### 5.4 차트

| 차트 타입 | 용도 |
|----------|------|
| 도넛 차트 | 비중 표시 (자산군별, 소유자별) |
| 프로그레스 바 | 단순 비율 비교 (위험도별) |
| 라인 차트 | 추이 (2단계에서 히스토리) |

```tsx
// 도넛 차트 컬러 (primary 계열 + 보조색)
const CHART_COLORS = [
  'var(--primary)',    // 브랜드 컬러
  '#03B26C',           // 초록
  '#FF9F00',           // 주황
  '#F04452',           // 빨강
  '#8B95A1',           // 회색
];
```

## 7. 주요 화면 가이드

### 7.1 홈 (`/home`)

**역할**: 로그인 후 첫 화면, 자산 현황 요약 및 빠른 액션 제공

```
구성:
1. 인사말 + 총 자산 (가장 크게)
2. 빠른 액션 버튼 (자산 기록, 분석, 설정)
3. 자산 유형별 현황 카드 (주식, 현금, 부동산, 기타)
4. 환율 정보 (작게)

원칙:
- 총 자산이 가장 눈에 띄어야 함
- 자산 유형 카드는 클릭 시 해당 유형 페이지로 이동
- "준비 중" 유형은 비활성화 상태로 표시
```

### 7.2 자산 메인 (`/assets`)

**역할**: 자산 유형 선택 허브, 전체 자산 요약

```
구성:
1. 총 자산 요약
2. 자산 유형별 카드 (클릭 시 해당 유형으로 이동)
   - 주식/ETF: 보유 종목 수, 수익률, [보유 현황] [거래 내역] 바로가기
   - 현금/예적금: 준비 중
   - 부동산: 준비 중
   - 기타: 준비 중
3. [+ 자산 기록] 플로팅 버튼

원칙:
- 자산 유형을 먼저 선택하는 계층 구조
- 각 유형별로 요약 정보 제공
- 확장 시 새 유형 카드 추가만으로 대응
```

### 7.3 대시보드 메인 (`/dashboard`)

**역할**: 분석 메인 허브, 핵심 지표 요약 및 분석 유형 선택

```
구성:
1. 총 자산 / 총 수익률 카드 (가장 크게)
2. 분석 유형 선택 카드
   - 자산 유형별: 주식, 현금(준비중), 부동산(준비중), 기타(준비중)
   - 비중 분석: 소유자별/위험도별/자산군별
3. 환율 정보 (작게)

원칙:
- 핵심 지표가 가장 먼저 눈에 들어와야 함
- 분석 유형 카드는 클릭 시 해당 분석 페이지로 이동
- "준비 중" 유형은 비활성화 상태로 표시
```

### 7.3.1 자산 유형별 분석 (`/dashboard/stocks` 등)

**역할**: 특정 자산 유형에 대한 상세 분석

```
구성:
1. 해당 자산 총액 / 수익률
2. 종목별 비중 차트 (도넛)
3. 종목별 상세 목록
4. 환율 정보 (해외 자산 시)

원칙:
- 해당 자산 유형의 전체 비중, 종목별 상세 제공
- 목록에서 종목 클릭 시 보유 현황 페이지로 이동
```

### 7.3.2 비중 분석 (`/dashboard/breakdown`)

**역할**: 다양한 기준으로 자산 비중 분석 (탭으로 전환)

```
구성:
1. 탭 메뉴: [소유자별] [위험도별] [자산군별]
2. 선택된 기준 비중 차트 (도넛)
3. 기준별 상세 내역 목록

원칙:
- 탭 전환 시 페이지 이동 없이 차트/목록만 갱신
- URL은 쿼리 파라미터로 탭 상태 유지 (예: ?tab=owner)
- 같은 레이아웃, 다른 데이터 시각화
```

### 7.4 주식 보유 현황 (`/assets/stock/holdings`)

**역할**: 주식/ETF 보유 종목 목록

```
구성:
1. 상단 탭: [보유 현황] [거래 내역]
2. 필터/정렬 옵션
3. 종목 리스트 (카드 또는 테이블)
4. [+ 거래 추가] 버튼

원칙:
- 테이블보다 카드 리스트 (모바일 친화)
- PC에서는 테이블 뷰 옵션
- 정렬/필터는 상단에
- 종목명 + 수익률이 핵심 정보
```

### 7.5 주식 거래 등록 (`/assets/stock/transactions/new`)

**역할**: 매수/매도 기록 입력

```
구성:
1. 거래 유형 선택 (매수/매도)
2. 종목 검색
3. 수량, 단가, 거래일 입력
4. 메모 (선택)
5. 저장 버튼

원칙:
- 한 화면에 모든 필드 (스크롤 최소화)
- 필수 입력만 먼저, 선택 입력은 접힘
- 저장 버튼은 항상 보이게 (sticky)
- 실시간 유효성 검사
```

## 8. 접근성

### 8.1 필수 사항

| 항목 | 기준 |
|------|------|
| 색상 대비 | 4.5:1 이상 (WCAG AA) |
| 포커스 표시 | 키보드 탐색 시 명확한 아웃라인 |
| 대체 텍스트 | 아이콘에 aria-label |
| 터치 영역 | 최소 44x44px |

### 8.2 스크린 리더

```tsx
// 숫자에 의미 부여
<span aria-label="총 자산 1억 2천 5백만원">
  ₩125,430,000
</span>

// 차트 대체 설명
<div role="img" aria-label="자산 비중: 주식 65%, 채권 20%, 현금 15%">
  <PieChart />
</div>
```
