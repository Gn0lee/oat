# DESIGN.md

> UI/UX 원칙 및 디자인 가이드

## TL;DR

- **토스 스타일 참고** - 큰 숫자, 충분한 여백, 카드 기반, 직관적 흐름
- **데이터 중심** - 핵심 숫자가 가장 먼저 눈에 들어와야 함
- **반응형** - 모바일(하단 탭 + 앱형 헤더), PC(사이드바 + breadcrumb top bar)
- **컬렉션 UI 우선** - 사용자 화면에서 테이블보다 카드, 리스트, 캘린더, 타임라인, 선택-상세 구조를 우선 검토
- **컬러** - 인디고 블루 기반, 상승 빨강/하락 파랑 (한국식)
- **다크모드** - MVP 제외, 2단계에서 추가

## 1. 디자인 원칙

### 1.1 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **데이터 중심** | 숫자와 핵심 정보가 가장 먼저 눈에 들어와야 함 |
| **단순함** | 한 화면에 하나의 목적, 불필요한 요소 제거 |
| **직관적 흐름** | 설명 없이도 다음 행동을 알 수 있어야 함 |
| **신뢰감** | 금융 서비스답게 안정적이고 깔끔한 인상 |

### 1.2 토스에서 배울 점

```
✅ 큰 숫자, 작은 레이블
✅ 충분한 여백 (빽빽하지 않게)
✅ 카드 기반 정보 그룹화
✅ 부드러운 전환 애니메이션
✅ 중요한 것만 색으로 강조
```

## 2. 레이아웃

### 2.1 서비스 레이아웃 구조

모바일과 PC는 같은 헤더를 늘려 쓰지 않습니다. 모바일은 앱처럼 화면 성격에 따라 상단 affordance를 바꾸고, PC는 좌측 사이드바와 콘텐츠 상단 breadcrumb로 위치를 안내합니다.

**모바일 Top-Level Screen**
```
┌─────────────────────────────────────────┐
│  Service Header (oat 로고)               │
├─────────────────────────────────────────┤
│                                         │
│              Main Content               │
│        (큰 페이지 제목 반복 없음)          │
│                                         │
├─────────────────────────────────────────┤
│  Bottom Navigation                      │
└─────────────────────────────────────────┘
```

**모바일 Child/Task Screen**
```
┌─────────────────────────────────────────┐
│  ←  화면 제목                    (X)     │
├─────────────────────────────────────────┤
│                                         │
│              Main Content               │
│        (큰 페이지 제목 반복 없음)          │
│                                         │
└─────────────────────────────────────────┘
```

**PC**
```
┌──────────────┬──────────────────────────┐
│ oat 로고      │  Breadcrumb               │
│              ├──────────────────────────┤
│  Sidebar     │                          │
│  Navigation  │       Main Content        │
│              │  (큰 페이지 제목 반복 없음) │
└─────────────────────────────────────────┘
```

### 2.2 반응형 브레이크포인트

| 이름 | 크기 | 대상 |
|------|------|------|
| `sm` | 640px | 모바일 (기본) |
| `md` | 768px | 태블릿 |
| `lg` | 1024px | 작은 데스크톱 |
| `xl` | 1280px | 데스크톱 |

```typescript
// Tailwind 사용 예시
<div className="px-4 md:px-6 lg:px-8">
  <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
```

### 2.3 네비게이션과 서비스 헤더

**모바일**: 하단 탭 네비게이션 + 화면 성격별 Service Header
```
[홈]  [가계부]  [자산]  [설정]
```

- Top-level 화면(`/home`, `/ledger`, `/assets`, `/settings`)은 로고형 Service Header를 사용하고 본문 큰 제목을 반복하지 않습니다.
- Child 화면은 `뒤로가기 + 화면 제목` 앱 바를 사용합니다.
- Task 화면은 필요 시 우상단 `X`를 제공합니다. `X`는 이전 화면이 아니라 작업 흐름을 닫고 안정적인 상위 목적지로 나가는 Close Action입니다.

**PC**: 사이드 네비게이션 + 콘텐츠 top bar breadcrumb
```
┌──────────────┐
│ oat 로고      │
├──────────────┤
│ 홈           │ → /home
│ 가계부        │ → /ledger
│ 자산         │ → /assets
│ 설정         │ → /settings
└──────────────┘
```

- 콘텐츠 영역 상단의 흰색 bar 좌측에 breadcrumb를 둡니다.
- PC 본문에서도 큰 페이지 제목을 반복하지 않습니다.
- 사이드바는 MVP에서 1차 메뉴만 유지할 수 있습니다. 계층형 사이드바는 후속 개선 방향입니다.
- 사이드바 상단은 일단 oat 로고만 둡니다. 가구/워크스페이스 identity는 별도 제품 개념으로 설계될 때 추가합니다.

**자산 페이지 내부**: 유형별 탭 또는 카드
```
/assets
├── 전체 보유 현황 → /assets/total/holdings
├── 주식/ETF      → /assets/stock/...
├── 현금 (준비중)  → /assets/cash/...
├── 부동산 (준비중) → /assets/real-estate/...
└── 기타 (준비중)  → /assets/other/...
```

## 3. 컬러

### 3.1 컬러 선정 원칙

| 원칙 | 설명 |
|------|------|
| 신뢰감 | 금융 서비스에 어울리는 안정적인 톤 |
| 현대적 | 올드하거나 투박하지 않은 느낌 |
| 절제 | 색상은 최소화, 강조점에만 사용 |

**추천 계열**
- 블루/인디고 계열: 신뢰, 안정
- 그린/틸 계열: 성장, 금융
- 퍼플 계열: 현대적, 프리미엄

### 3.2 컬러 구조

```css
:root {
  /* Primary - 인디고 블루 (신뢰감, 현대적) */
  --primary: #4F46E5;        /* Tailwind indigo-600 */
  --primary-dark: #4338CA;   /* indigo-700 */
  --primary-light: #EEF2FF;  /* indigo-50 */
  
  /* Neutral - 텍스트, 배경 */
  --gray-900: #191F28;       /* 메인 텍스트 */
  --gray-700: #4E5968;       /* 서브 텍스트 */
  --gray-500: #8B95A1;       /* 비활성 텍스트 */
  --gray-300: #D1D6DB;       /* 보더 */
  --gray-100: #F2F4F6;       /* 배경 */
  --white: #FFFFFF;
  
  /* Semantic - 의미 기반 */
  --positive: #F04452;       /* 상승 (빨강) */
  --negative: #3182F6;       /* 하락 (파랑) */
  --warning: #FF9F00;
  --success: #03B26C;
}
```

### 3.3 컬러 사용 규칙

| 용도 | 컬러 |
|------|------|
| 메인 텍스트 | `gray-900` |
| 보조 텍스트 | `gray-700` |
| 비활성/힌트 | `gray-500` |
| 주요 액션 버튼 | `primary` |
| 수익 상승 | `positive` (빨강) |
| 수익 하락 | `negative` (파랑) |
| 배경 | `gray-100` 또는 `white` |
| 카드 배경 | `white` |

### 3.4 다크모드 (2단계)

MVP에서는 라이트모드만 지원. 2단계에서 다크모드 추가 예정.

```css
/* 2단계에서 추가 */
[data-theme="dark"] {
  --gray-900: #FFFFFF;
  --gray-100: #191F28;
  /* ... */
}
```

## 5. 컴포넌트 가이드

### 5.1 카드

정보를 그룹화하는 기본 단위

```tsx
<div className="bg-white rounded-2xl p-5 shadow-sm">
  {/* 내용 */}
</div>
```

| 속성 | 값 |
|------|-----|
| 배경 | `white` |
| 모서리 | `rounded-2xl` (16px) |
| 패딩 | `p-5` (20px) |
| 그림자 | `shadow-sm` |

### 5.2 버튼

```tsx
// Primary
<Button className="w-full bg-primary text-white rounded-xl py-3">
  자산 추가하기
</Button>

// Secondary
<Button variant="outline" className="rounded-xl">
  취소
</Button>
```

| 타입 | 용도 |
|------|------|
| Primary | 주요 액션 (저장, 추가, 확인) |
| Secondary/Outline | 보조 액션 (취소, 뒤로) |
| Ghost | 덜 중요한 액션 |

### 5.3 입력 필드

```tsx
<div className="space-y-2">
  <label className="text-sm text-gray-700">종목명</label>
  <Input 
    className="rounded-xl border-gray-300 focus:border-primary"
    placeholder="종목을 검색하세요"
  />
</div>
```

### 5.4 차트

| 차트 타입 | 용도 |
|----------|------|
| 도넛 차트 | 비중 표시 (자산군별, 소유자별) |
| 프로그레스 바 | 단순 비율 비교 (위험도별) |
| 라인 차트 | 추이 (2단계에서 히스토리) |

```tsx

  'var(--primary)',    // 브랜드 컬러
  '#03B26C',           // 초록
```

### 5.5 컬렉션 UI

사용자가 금융 기록이나 자산을 훑어보는 화면은 테이블을 기본값으로 두지 않습니다. PC와 모바일 모두 데이터 성격에 맞는 컬렉션 UI를 우선 검토합니다.

| 데이터 성격 | 권장 UI | 예시 |
|------------|---------|------|
| 날짜가 핵심인 기록 | 캘린더, 날짜별 목록, 타임라인 | 주식 거래 내역, 가계부 기록 |
| 비교가 필요한 보유 항목 | 넓은 리스트, 요약 카드, 선택-상세 | 주식 보유 종목 |
| 설정/관리 항목 | compact list, grouped card | 종목 설정, 계좌, 결제수단 |
| 사람/초대 상태 | 사람 중심 리스트, 상태별 섹션 | 가구 구성원, 대기 중 초대 |
| 월별/기간별 요약 | metric list, timeline, 현재 기간 강조 | 월별 상세 |

원칙:
- 테이블 컬럼을 그대로 카드에 옮기지 않습니다.
- 각 항목은 대표값 1개, 보조 정보 2-4개, 필요한 액션으로 재구성합니다.
- 정렬/필터는 테이블 헤더 클릭이 아니라 상단 control로 제공합니다.
- PC의 넓은 화면은 더 많은 컬럼보다 그룹핑, 비교, 선택-상세 영역에 사용합니다.
- 내부 table primitive나 TanStack Table 의존성 제거는 별도 정리 대상입니다.

화면별 기본 방향:

| 화면 | 권장 구성 |
|------|----------|
| 주식 거래 내역 | 월 캘린더 또는 날짜 summary, 선택 날짜 거래 목록, 거래 상세/수정 흐름 |
| 주식 보유 현황 | 종목 리스트/카드, 투자금/수량/평균가 중심, 소유자/계좌/시장 badge |
| 종목 설정 | 종목명/티커 중심 compact list, 시장/위험도 badge, 편집 액션 |
| 계좌 | 은행 계좌/투자 계좌 그룹, 기관/유형/계좌번호/기본 계좌 상태 |
| 결제수단 | 이름/유형/카드사/끝자리/연결 계좌/기본 여부 |
| 가구 구성원 | 구성원과 대기 중 초대를 분리한 사람 중심 리스트 |
| 월별 상세 | 월, 수입, 지출, 저축률을 한 묶음으로 보여주는 timeline 또는 metric list |

## 7. 주요 화면 가이드

### 7.1 홈 (`/home`)

**역할**: 로그인 후 첫 화면, 자산 현황 요약 및 빠른 액션 제공

```
구성:
1. 인사말 + 총 자산 (가장 크게)
2. 빠른 액션 버튼 (자산 기록, 분석, 설정)
3. 자산 유형별 현황 카드 (주식, 현금, 부동산, 기타)
4. 환율 정보 (작게)

원칙:
- 총 자산이 가장 눈에 띄어야 함
- 자산 유형 카드는 클릭 시 해당 유형 페이지로 이동
- "준비 중" 유형은 비활성화 상태로 표시
```

### 7.2 자산 메인 (`/assets`)

**역할**: 자산 유형 선택 허브, 전체 자산 요약

```
구성:
1. 총 자산 요약
2. 자산 유형별 카드 (클릭 시 해당 유형으로 이동)
   - 주식/ETF: 보유 종목 수, 수익률, [보유 현황] [거래 내역] 바로가기
   - 현금/예적금: 준비 중
   - 부동산: 준비 중
   - 기타: 준비 중
3. [+ 자산 기록] 플로팅 버튼

원칙:
- 자산 유형을 먼저 선택하는 계층 구조
- 각 유형별로 요약 정보 제공
- 확장 시 새 유형 카드 추가만으로 대응
```

### 7.3 전체 자산 분석 (`/assets/analysis`)

**역할**: 분석 메인 허브, 핵심 지표 요약 및 분석 유형 선택

```
구성:
1. 총 자산 / 총 수익률 카드 (가장 크게)
2. 분석 유형 선택 카드
   - 자산 유형별: 주식, 현금(준비중), 부동산(준비중), 기타(준비중)
   - 비중 분석: 소유자별/위험도별/자산군별
3. 환율 정보 (작게)

원칙:
- 핵심 지표가 가장 먼저 눈에 들어와야 함
- 분석 유형 카드는 클릭 시 해당 분석 페이지로 이동
- "준비 중" 유형은 비활성화 상태로 표시
```

### 7.3.1 자산 유형별 분석 (`/assets/stock/analysis` 등)

**역할**: 특정 자산 유형에 대한 상세 분석

```
구성:
1. 해당 자산 총액 / 수익률
2. 종목별 비중 차트 (도넛)
3. 종목별 상세 목록
4. 환율 정보 (해외 자산 시)

원칙:
- 해당 자산 유형의 전체 비중, 종목별 상세 제공
- 목록에서 종목 클릭 시 보유 현황 페이지로 이동
```

### 7.3.2 비중 분석 (`/assets/analysis/by-owner`, `/assets/analysis/by-risk`, `/assets/analysis/by-asset-type`)

**역할**: 다양한 기준으로 자산 비중 분석

```
구성:
1. 소유자별/위험도별/자산군별 분석 페이지
2. 선택된 기준 비중 차트 (도넛)
3. 기준별 상세 내역 목록

원칙:
- 탭 전환 시 페이지 이동 없이 차트/목록만 갱신
- URL은 쿼리 파라미터로 탭 상태 유지 (예: ?tab=owner)
- 같은 레이아웃, 다른 데이터 시각화
```

### 7.4 주식 보유 현황 (`/assets/stock/holdings`)

**역할**: 주식/ETF 보유 종목 목록

```
구성:
1. 상단 탭: [보유 현황] [거래 내역]
2. 필터/정렬 옵션
3. 종목 리스트 또는 카드
4. [+ 거래 추가] 버튼

원칙:
- 사용자-facing 테이블보다 컬렉션 UI를 기본으로 함
- PC에서는 넓은 리스트 또는 선택-상세 구조를 우선 검토
- 정렬/필터는 상단에
- 종목명, 투자금, 수량, 평균 매수가가 핵심 정보
```

### 7.4.1 주식 거래 내역 (`/assets/stock/transactions`)

**역할**: 주식/ETF 매수·매도 기록을 날짜 흐름으로 조회

```
구성:
1. 월 이동 + 필터 + [+ 거래 추가]
2. 월간 캘린더 또는 날짜별 요약
3. 선택 날짜 거래 목록
4. 거래 상세/수정/삭제 흐름

원칙:
- 날짜가 핵심 축이므로 단순 row 목록보다 캘린더/날짜별 목록을 우선함
- PC는 캘린더/날짜 요약과 선택 날짜 목록을 함께 보여줄 수 있음
- 모바일은 캘린더 -> 날짜별 목록 -> 상세/수정 흐름으로 분리함
- 거래 item은 종목명, 매수/매도, 금액을 우선하고 수량/단가/소유자는 보조 정보로 둠
- API는 우선 기존 거래 목록 데이터를 사용하고, 데이터 양 문제가 확인될 때 월 단위 서버 필터를 검토함
```

### 7.5 주식 거래 등록 (`/assets/stock/transactions/new`)

**역할**: 매수/매도 기록 입력

```
구성:
1. 거래 유형 선택 (매수/매도)
2. 종목 검색
3. 수량, 단가, 거래일 입력
4. 메모 (선택)
5. 저장 버튼

원칙:
- 한 화면에 모든 필드 (스크롤 최소화)
- 필수 입력만 먼저, 선택 입력은 접힘
- 저장 버튼은 항상 보이게 (sticky)
- 실시간 유효성 검사
```
